На цьому кроці потрібно написати та перевірити SQL-запит. Спочатку коротко описується структура та особливості запиту, наводиться приклад. А потім формулюється завдання, для якого потрібно реалізувати запит.

Створення таблиці

Для створення таблиці використовується SQL-запит. У ньому зазначається, яку таблицю створюють, з яких атрибутів (полів) вона складається та який тип даних має кожне поле. За потреби вказується опис полів (ключове поле тощо). Його структура:

  • ключове слово CREATE TABLE
  • ім’я створюваної таблиці
  • відкрита кругла дужка «(»
  • назва поля та його опис, що включає тип поля та інші необов’язкові характеристики
  • кома
  • назва поля та його опис
  • ...
  • закрита дужка «)»

Приклад. Створимо таблицю genre такої структури:

Поле Тип, опис
genre_id INT PRIMARY KEY AUTO_INCREMENT
name_genre VARCHAR(30)

Запит:

CREATE TABLE genre(
    genre_id INT PRIMARY KEY AUTO_INCREMENT, 
    name_genre VARCHAR(30)
);

Створена таблиця — порожня.

Рекомендації щодо запису SQL-запиту

  • SQL — нечутлива до регістру мова (наприклад, CREATE і create — одне й те саме ключове слово).
  • Ключові слова SQL та типи даних рекомендується писати великими літерами.
  • Назви таблиць і полів — малими літерами.
  • SQL-запит можна писати на кількох рядках.
  • У кінці SQL-запиту ставиться крапка з комою ; (хоча, якщо ви пишете один запит, це не обов’язково).

Завдання

Сформулюйте SQL-запит для створення таблиці book, введіть його у вікно коду (розташоване нижче) та натисніть кнопку Відправити для перевірки. Структура таблиці book:

Поле Тип, опис
book_id INT PRIMARY KEY AUTO_INCREMENT
title VARCHAR(50)
author VARCHAR(30)
price DECIMAL(8, 2)
amount INT

Пояснення

Під час запису зберігайте порядок слідування полів.
