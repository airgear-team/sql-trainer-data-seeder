Завдання

Завершити етап «Оплата» для замовлення з номером 5, вставивши у стовпець date_step_end дату 13.04.2020, та розпочати наступний етап («Упаковка»), вказавши у стовпці date_step_beg для цього етапу ту ж дату.

Необхідно реалізувати два запити — один для завершення поточного етапу, інший — для початку наступного. Запити мають бути універсальними: їх можна буде використати для будь-якого етапу, змінивши тільки назву поточного етапу. Для прикладу використаємо етап «Оплата».

Пояснення

У таблицю step усі необхідні етапи занесено послідовно. Якщо поточний етап — «Оплата» (його id = 1), то наступний етап — «Упаковка» матиме id = 2. Тому у запиті, що оновлює дату початку наступного етапу, можна використати вкладений запит: 

SELECT step_id + 1 
FROM step
WHERE name_step = 'Оплата'

Пов’язані кроки
  • оновлення даних;
  • умова відбору (крок, крок, крок);
  • вкладені запити в умовах відбору.

Реалізувати два універсальних запити, змінюючи тільки назву поточного етапу (у прикладі — 'Оплата').
