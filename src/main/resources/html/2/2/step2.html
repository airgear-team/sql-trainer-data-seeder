Оператор LEFT і RIGHT OUTER JOIN

Оператор зовнішнього з’єднання LEFT OUTER JOIN (можна писати просто LEFT JOIN) поєднує дві таблиці. Порядок таблиць важливий, оскільки цей оператор не є симетричним. 

SELECT
    ...
FROM
    таблиця_1 LEFT JOIN таблиця_2
    ON умова
    ...

Результат запиту формується так:

  • додаються всі записи, що підпадають під внутрішнє з’єднання INNER JOIN між першою та другою таблицями;
  • додаються всі рядки з першої таблиці, які не мають відповідних записів у другій таблиці; для таких записів усі поля другої таблиці будуть NULL.

Оператор RIGHT JOIN працює аналогічно, але в другому кроці ролі таблиць міняються місцями — усі рядки з другої таблиці включаються у результат, навіть якщо немає відповідності в першій.

Приклад

Вивести назви всіх книг кожного автора. Якщо в автора немає книг — виводити NULL замість назви.

Запит:

SELECT name_author, title 
FROM author LEFT JOIN book
     ON author.author_id = book.author_id
ORDER BY name_author;

Пояснення

При використанні LEFT JOIN назви книг і інші поля з таблиці book для жанрів, яких немає в книгах, будуть мати значення NULL. Щоб перевірити такі значення, потрібно використовувати IS NULL (а не = NULL).
